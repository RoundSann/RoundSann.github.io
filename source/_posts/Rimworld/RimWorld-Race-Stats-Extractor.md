# RimWorld-Race-Stats-Extractor (边缘世界种族数据提取器)

 > **⚠AI制作屎山警告⚠!**

一个用于RimWorld的py小脚本,可以自动拉取工作目录下的所有mod文件里的外星人XML定义数据并生成表格,帮助你快速认知一个种族在XML定义上擅长什么工作,不擅长什么工作.

原始数据版仅生成原始XML表格(相当于旧版本).普通版则除了原始XML表格,还能应用**基因补丁**,计算**基因影响**,输出一个种族的**特性偏好**.并生成表格.

另外,整体表格仅供参考,肯定有不兼容的情况,建议你校对一遍为妙

## 使用方式

 1. **配置路径**：  
    打开脚本文件, 找到配置区域, 修改 `MOD_FOLDER_PATH` 为你的工作路径,脚本会抓取工作路径下的所有XML文件中符合要求的部分.  
    翻译成人话就是你把想被提取的mod都丢到这个文件夹下就行.或者直接改成你的steam workshop文件夹也行,但是你mod多的话处理可能会比较慢
    ```python
    # 例如
    MOD_FOLDER_PATH = "./Mods"
    ```
    特别地,如果你想应用原版的基因补丁,**需要让生物科技DLC也在工作目录下**,否则原版基因的效果不会被计算

 2. **调整配置 (可选)**：  
    参考后面配置部分

 3. **运行脚本**:  
    不会运行py程序?那问问豆包/Deepseek/GPT/Gemini.

 4. **查看结果**：  
    程序运行结束后, 会在目录下生成 `Race输出列表.md`. 你可以使用任何支持 Markdown 的编辑器（如 VSCode, Typora, Obsidian）查看, 或直接粘贴到在线表格转换工具中.  
    在线表格编辑可使用[TableConvert](https://tableconvert.com/zh-cn/markdown-to-markdown).这个网站还支持markdown转excel


## 主要功能及其接口介绍
### 开关及接口
 -  `MOD_FOLDER_PATH`: 工作目录,只会提取工作目录下的所有mod的xml文件.  
    特别地如果你想应用原版的基因补丁,**需要让生物科技DLC也在工作目录下**,否则原版基因的效果不会被计算
 -  `MODE`: 输出时对数值覆盖的模式选择
     -  `RAW`: 输出原始数据,XML里写的啥程序就输出啥,不做任何更改,XML里未定义的值会被输出为`-`
     -  `NON_DEFAULT`: 隐藏等同于默认值的数据,将所有与默认值相等的数据输出为`-*`
     -  `ALL_DATA`: 将所有未定义的数值使用默认值填充并输出
 -  `GENE_CALCULATION_MODE`: 基因补丁的模式
     -  `NO_GENE`: 无视基因补丁仅显示默认数据
     -  `GENE_RAW`: 使用基因补丁数据覆盖掉默认数据  
        举个例子: 绮罗的数值会因为你是否启用绮罗基因而改变,未启用绮罗基因时,其瞄准时间为110%,而在启用时,则为100%,同时基因效果为$\times$ 110%.当你使用该选项时,输出的表格将会填入补丁数值 100%, 而不是原始定义 110%
     -  `GENE_ADDITION`: 使用基因补丁数据覆盖掉默认数据同时应用基因效果  
        还是上面那个例子,在此模式下将会把基因效果也算上去,显示成 $100\%\times 1.1$.  
        此处不直接计算结果是因为可能会涉及一些 先乘后加 或 先加后乘 之类的运算优先级问题,导致叠叠乐.  
        例如,某种族原始定义全局工作速度为150%,基因补丁定义为100%,基因效果为全局工作速度$\times$ 150%,另外其有特性使得全局工作速度 +30%.那么此时他在应用基因补丁的效果为 $(100\%+30\%)\times 150\%=195\%$.而在不应用基因补丁的效果为 $150\%+30\%=180\%$ 二者显然不等
     -  `GENE_FULL`: 输出 默认数据 同时输出 应用了基因效果的基因补丁数值  
        例如,在绮罗的例子中,在该模式下最终其会显示成 $110\%/100\%\times 1.1$.
 -  `ENABLE_AUTO_SPLIT`: 是否开启表格自动分割
 -  `SPLIT_ROW_LIMIT`: 若开启自动分割,每张表格最多显示多少行(不包括表头行)
 -  `SPLIT_COL_LIMIT`: 若开启自动分割,每张表格最多显示多少列(不包括属性名列和默认值列)
 -  `ADD_DEFAULT_RACE_COLUMN`: 是否输出默认值参考列,若开启,则在最左侧输出一列昵称为"智人种",数据填充均为默认值的列作为数值的参考
 -  `ONLY_OUTPUT_TRANSLATED_RACES`: 种族输出名单,即是否仅输出在`RACE_NAME_TRANSLATION`变量里命名了的种族
 -  `ENABLE_FULL_DATA_PANEL`: 是否输出完整表格.如果输出,则会输出所有在`PROPERTY_MAP`有定义的行所组成的表格,超级大所以一般不输出
 -  `ENABLE_MAIN_PANEL`: 是否输出主要表格,此表格仅包含主要需要注意的数据.如果输出,则会输出所有在`MAIN_PANEL_LIST`有定义的行所组成的表格,当然你也可以自己diy你更注意的项,只需更改`MAIN_PANEL_LIST`的内容即可,会按照顺序输出表格.
 -  `ENABLE_SUPPLEMENTARY_PANEL`: 是否输出不在主要表格内但又在完整表格内的数值(换言之,主要表格的补集),方便你观察除了主要表格以外还有什么变化.
 -  `ENABLE_CHANGES_PANEL`: 是否输出差异统计面板/变化面板.变化表格的意思是:如果某行中,有种族的某项数值与默认值不同,那么输出表格时就有该行,否则不输出该行.  
    例如,如果仅有A,B,C三个种族,且其全局工作速度都是100%,等同于默认值,那么最终输出的变化面板中就不会出现全局工作速度这一行.但如果其中有任意一个种族与默认值不同,则最终变化面板都会输出这一行.因为这一行有变化的数值了.变化面板兼容基因补丁,可以放心
 -  `ENABLE_UNCHANGED_PROPS`: 是否输出未变化的面板,方便你观察都有什么数值没有变化,也就是变化面板的补集.一般不开
 -  `ENABLE_CUSTOM_VIEWS`: 是否输出自定义的子表格,此项是给你自己DIY用的,你可以自己创建自己想要输出的表格,把你想要输出的项填写在变量`CUSTOM_VIEWS_CONFIG`中就能够自定义输出
 -  `ENABLE_TRAIT_DETAILS`: 是否输出一个种族的特性信息.特性信息包含强制的特性/禁止的特性/专属特性等.**未经过广泛验证,目前仅供参考**
 -  `ENABLE_XENOTYPE_MANIFEST`: 是否输出异种胚清单,如果输出,则会在文档的最后显示mod列表里一共有多少种异种胚基因组,此项功能是为了给美狐,蚁族,鼠族等有多套基因组的外星人指定其默认基因用的
 -  `ENABLE_RACE_XENOTYPE_BINDING_REPORT`: 是否输出当前种族所绑定的异种胚,如果输出,则会在文档的最后显示当前文档中,每个种族各自绑定了什么异种胚基因组,方便你确认是否有错误的基因绑定之类.

### 字典及作用
 -  `PROPERTY_MAP`: 属性映射表,把各项属性映射到XML的键里.特别地,躯干耐久,食物消耗和袭击点数这三项没有找到其对应的键,需要手动填充
 -  `RACE_NAME_TRANSLATION`: 种族翻译,原始输出的表格全都是def名形式的,如Kiiro_Race,此字典可以将def名翻译成对应的名称,同时由开关`ONLY_OUTPUT_TRANSLATED_RACES`控制是否仅输出在种族翻译里的种族.如果你不想输出某种族,注释掉它即可
 -  `DEFAULT_VALUES`: 默认值,此字典为大部分属性都添加了默认值,用于默认值参考列,同时也用于MODE中与默认值相同的数值被隐藏.没有默认值也能跑,但你可能会很蛋疼
 -  `PERCENTAGE_FIELDS`: 需要按百分比显示的数值,此字典内的所有属性都会以百分比显示,而不是以数字显示
 -  `MAIN_PANEL_LIST`: 主要面板,此字典的所有属性会按照顺序被主要面板进行输出
 -  `CUSTOM_VIEWS_CONFIG`: 自定义子面板,你可以在这个字典里diy自己想要的表格,会按照顺序进行输出
 -  `TRAIT_TRANSLATION`: 特性信息默认都是def名,此字典对def名进行翻译  
    特别地,一些特性可能会有多种"程度",例如Industriousness的1程度就是工作勤奋,Industriousness的2程度就是工作狂,Industriousness的-2程度就是工作懈怠.此字典支持如同 `'Industriousness-2':'工作懈怠'` 这样对不同程度的特性进行汉化  
    **内容部分未经验证仅供参考**
 -  `MANUAL_GENE_EFFECTS`: 基因补丁,本身程序只能识别由XML所定义的基因,但是有些基因是写在C里面或是给予特性实现的,此补丁就是把这种"基因"翻译成程序能识别到的方式.  
    例如,原版基因"心灵失聪"效果为为小人添加一个"心灵失聪"特性,其效果基本等同于直接-100%心灵敏感度,但是这个基因是赋予基因实现而不是写在XML定义里,所以输出的时候你就看不到这个效果,此时有了补丁,就能把它当成-100%心灵敏感度来看从而最终输出时显示在表格里
 -  `MANUAL_XENOTYPE_MAPPING`: 强制异种胚列表,可以将某个种族绑定某个基因,用于解决鼠族,美狐,蚁族这种一个种族有多套基因的,直接为种族强制绑定基因组是最好的选择.  
    同时开关中的`ENABLE_XENOTYPE_MANIFEST`(是否输出异种胚清单),`ENABLE_RACE_XENOTYPE_BINDING_REPORT`(是否输出当前种族所绑定的异种胚)就是为了让你知道都有哪些异种胚,当前种族都绑定了什么异种胚所添加的
